# 需求文档

## 1. 应用概述

### 1.1 应用名称
Pause

### 1.2 应用描述
Pause 是一个纯粹的个人单词学习工具，以纯前端网页形式运行。核心哲学为温柔、克制、不制造焦虑。采用 React + TypeScript + Tailwind CSS + shadcn/ui 技术栈，使用 localStorage 进行前端本地存储，通过 JSON 手动导出/导入实现数据同步。

## 2. 用户与使用场景

### 2.1 目标用户
需要学习和积累单词的个人用户，追求简洁、温和的学习体验。

### 2.2 核心使用场景
- 每日利用碎片时间进行单词学习与复习
- 通过阅读文章提取生词并学习
- 在写作时获取已学单词的建议
- 查看学习进度与打卡记录

## 3. 页面结构与功能说明

### 3.1 页面结构

```
├── 首页
├── 单词学习页
├── 复习模式页
├── 写作沙盒页
├── 一键提取生词页
├── 词库管理页
├── 语言花园页
└── 设置页
```

### 3.2 首页

#### 3.2.1 问候语
- 展示问候语

#### 3.2.2 20分钟倒计时圆环
- 展示20分钟倒计时圆环
- 用户可启动/暂停倒计时

#### 3.2.3 今日任务概要
- 展示今日学习任务概要信息

#### 3.2.4 底部导航
- 提供导航入口，可跳转至其他主要页面

### 3.3 单词学习页

#### 3.3.1 核心卡片流
- 正面展示：单词、音标、发音
- 下滑操作：展示释义、例句
- 左滑操作：展示同义词
- 右滑操作：展示形近词

#### 3.3.2 单词数据获取
- 使用 Free Dictionary API 获取单词数据

### 3.4 复习模式页

#### 3.4.1 例句填空
- 展示例句，用户填写缺失单词

#### 3.4.2 磁吸拼词
- 提供字母块，用户拖拽拼出单词

#### 3.4.3 消失的字母
- 展示部分字母缺失的单词，用户补全

### 3.5 写作沙盒页

#### 3.5.1 书写区
- 提供极简书写区域，用户可输入文本

#### 3.5.2 单词建议
- 基于用户已学单词，提供写作建议

### 3.6 一键提取生词页

#### 3.6.1 文章粘贴
- 用户粘贴文章内容

#### 3.6.2 生词提取
- 提取文章中的生词
- 比对本地词库，识别未学单词

#### 3.6.3 语境卡生成
- 自动生成包含语境的单词卡片

### 3.7 词库管理页

#### 3.7.1 语言分组
- 按语言对单词进行分组展示

#### 3.7.2 状态筛选
- 按学习状态筛选单词（如：未学、学习中、已掌握）

#### 3.7.3 搜索
- 支持单词搜索

#### 3.7.4 来源标签
- 展示单词来源标签

### 3.8 语言花园页

#### 3.8.1 进度可视化
- 使用植物成长形式展示学习进度

#### 3.8.2 打卡统计
- 展示打卡记录与统计数据

### 3.9 设置页

#### 3.9.1 数据导出/导入
- 支持将数据导出为 JSON 文件
- 支持导入 JSON 文件恢复数据

#### 3.9.2 每日时长设置
- 用户可设置每日学习时长目标

#### 3.9.3 主题切换
- 支持切换应用主题

## 4. 业务规则与逻辑

### 4.1 遗忘曲线算法
- 使用简化 SM-2 算法计算单词下次复习时间
- 根据用户复习表现调整复习间隔

### 4.2 数据存储
- 所有数据存储在浏览器 localStorage 中
- 数据包括：单词信息、学习记录、复习计划、用户设置

### 4.3 数据同步
- 通过手动导出 JSON 文件实现数据备份
- 通过导入 JSON 文件实现数据恢复或迁移

### 4.4 词典接口调用
- 调用 Free Dictionary API 获取单词的音标、释义、例句、同义词、形近词等信息

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| Free Dictionary API 调用失败 | 提示用户网络异常，稍后重试 |
| localStorage 存储空间不足 | 提示用户清理数据或导出备份 |
| 导入 JSON 文件格式错误 | 提示用户文件格式不正确 |
| 用户粘贴空白文章 | 提示用户输入有效内容 |
| 提取生词时无新词 | 提示用户文章中无新生词 |

## 6. 设计规范

### 6.1 配色方案
- 背景色：#FAF8F5（暖白）
- 强调色：#9CAF88（鼠尾草绿）

### 6.2 字体规范
- 单词展示：使用衬线体（Georgia）
- UI 界面：使用无衬线体（Inter）

### 6.3 响应式设计
- 手机优先设计（max-width 480px）

## 7. 验收标准

1. 用户打开首页，查看今日任务概要
2. 用户进入单词学习页，通过卡片流学习单词（正面查看单词/音标/发音，下滑查看释义/例句，左滑查看同义词，右滑查看形近词）
3. 用户进入复习模式页，完成例句填空、磁吸拼词或消失的字母练习
4. 用户进入词库管理页，查看已学单词并按语言分组或状态筛选
5. 用户进入语言花园页，查看学习进度可视化与打卡统计
6. 用户进入设置页，导出数据为 JSON 文件
